Task 4: Fix log-scale rendering — Evidence
==========================================

BEFORE: log-scale SSIM = 0.9012 (worst example)
AFTER:  log-scale SSIM = 0.9721 (above 0.95 target)

Root Cause:
- trans-scale field was initialized to identity and NEVER incorporated into data→display pipeline
- %update-trans-data ignored trans-scale entirely
- axes-set-yscale/axes-set-xscale did not set trans-scale
- %setup-transforms reset trans-scale to identity on every draw
- log-transform only transformed X axis, not Y axis
- Autoscale margin applied in linear space created negative Y values for log scale

Changes Made:
1. src/primitives/scale-transforms.lisp:
   - Added log-y-transform class (log on Y, pass X through)
   - Added inverted-log-y-transform class  
   - Added transform-path methods for log-transform, log-y-transform, and their inverses

2. src/packages.lisp:
   - Exported log-y-transform, inverted-log-y-transform and their accessors

3. src/containers/axes-base.lisp:
   - %setup-transforms: preserve existing trans-scale (don't reset to identity)
   - %update-trans-data: incorporate trans-scale in pipeline
     Pipeline: data → trans-scale → scaled-viewLim→unit → transAxes → display
     For identity trans-scale (linear), reduces to original pipeline
   - axes-set-yscale: set trans-scale to log-y-transform, recompute view limits with log-space margins
   - axes-set-xscale: set trans-scale to log-transform, recompute view limits with log-space margins
