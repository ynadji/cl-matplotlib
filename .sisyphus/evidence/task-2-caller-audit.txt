=== IMPACT ANALYSIS ===

The 5 broken callers all use the same incorrect pattern:
  (multiple-value-list (mpl.colors:to-rgba color))

This pattern produces: ((#(r g b a))) instead of (r g b a)

When this is passed to functions expecting (r g b a), it causes:
  - Type errors when trying to access elements
  - Incorrect color rendering
  - Potential crashes in graphics context creation

CORRECT PATTERNS (from working code):
  1. Direct vector access: (aref rgba 0), (aref rgba 1), etc.
  2. Conditional check: (if (vectorp rgba) (list (aref rgba 0) ...) ...)
  3. Store in variable: (let ((rgba (to-rgba ...))) (aref rgba 0) ...)

=== RECOMMENDED FIX PATTERN ===

Replace all broken instances with the pattern from backend-vecto.lisp:

  (let ((rgba (mpl.colors:to-rgba color-spec)))
    (if (vectorp rgba)
        (list (float (elt rgba 0) 1.0)
              (float (elt rgba 1) 1.0)
              (float (elt rgba 2) 1.0)
              (float (elt rgba 3) 1.0))
        (multiple-value-list rgba)))

This handles both vector returns (current behavior) and potential future
multiple-value returns (defensive programming).

Alternatively, simpler fix (if confident to-rgba always returns vector):

  (list (aref rgba 0) (aref rgba 1) (aref rgba 2) (aref rgba 3))

where rgba is bound from (mpl.colors:to-rgba color).
